# LunchPad iOS — Setup guide

## What this is

A Capacitor iOS shell that wraps the LunchPad web app (lunchpad.us).

On first launch the parent sees a native school-code entry screen. They type
their school's slug (e.g. `fsskitchen`) and the app navigates permanently to
`https://fsskitchen.lunchpad.us`. The web app handles all auth and ordering.

- **Bundle ID**: `us.lunchpad.app`
- **Apple ID for signing**: `faheed@live.com`
- **App name**: LunchPad

## Codemagic setup (one-time, ~20 min) ← CURRENT

### Step 1 — Push the repo to GitHub

In PowerShell from `C:\Faheed Code\lunchpad-ios`:

```powershell
git init
git remote add origin https://github.com/YOUR_USERNAME/lunchpad-ios.git
git branch -M main
git add .
git commit -m "Day 1: LunchPad iOS scaffold with school-code entry + Codemagic CI"
git push -u origin main
```

Create the GitHub repo first at https://github.com/new — name it `lunchpad-ios`, Private, no README.

---

### Step 2 — Create a Codemagic account and connect the repo

1. https://codemagic.io → sign up (use your GitHub account)
2. **Add application** → GitHub → select `lunchpad-ios`
3. When asked for workflow type → choose **`codemagic.yaml`**

---

### Step 3 — Register App Store Connect API key with Codemagic

**Create the API key:**
1. https://appstoreconnect.apple.com/access/integrations/api → **+**
2. Name: `Codemagic CI`, Access: **App Manager** → Generate
3. Download the `.p8` file immediately (only shown once)
4. Note the **Key ID** and **Issuer ID**

**Add to Codemagic:**
1. Codemagic → **Teams** → your team → **Integrations** → **Apple Developer Portal**
2. **Add** → Key ID, Issuer ID, upload `.p8`
3. Name it exactly: **`Faheed ASC Key`**

---

### Step 4 — Register the bundle ID and app in Apple

**Register bundle ID:**
1. https://developer.apple.com/account/resources/identifiers/list → **+**
2. App IDs → App → Continue
3. Description: `LunchPad`, Bundle ID: **Explicit** → `us.lunchpad.app`
4. Capabilities: enable **Sign In with Apple** and **Push Notifications** → Register

**Create the app in App Store Connect:**
1. https://appstoreconnect.apple.com/apps → **+** → New App
2. Platform: iOS, Name: `LunchPad`, Bundle ID: `us.lunchpad.app`, SKU: `lunchpad-ios`
3. Primary language: English (US)

---

### Step 5 — Trigger your first build

Push any change to `main`, or in Codemagic click **Start build** → select **LunchPad iOS — Build & TestFlight**.

- First build: ~20 min (CocoaPods install is slow on first run)
- Subsequent builds: ~10–12 min

Watch the build logs in real time — each step is shown clearly.

---

### Step 6 — Install on your iPhone via TestFlight

1. Install **TestFlight** on your iPhone from the App Store
2. App Store Connect → LunchPad → **TestFlight** → **Internal Testing** → add `faheed@live.com`
3. Accept the email invite on your iPhone → install LunchPad
4. Open app → type `fsskitchen` → tap **Open LunchPad** → you should land on the restaurant's ordering page

## Notes

- The `ios/` Xcode project is generated by Codemagic on every build (`npx cap add ios` if missing, then `npx cap sync ios`). You don't need a Mac to build.
- After getting a Mac, run `npm install && npx cap add ios && npx cap open ios` locally to open Xcode and iterate faster.
- `lbb-hotlunch-ios` is a separate project for the Medina Academy app — do not mix them.
